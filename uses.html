Uses

The hardware, terminal setup, and day-to-day tools behind shipping and running infrastructure — from local dev to production AWS accounts.

jordan@bastion:~/uses$ aws sts get-caller-identity
hardware editor & terminal cloud & aws iac & config ci/cd & containers monitoring & collab
~/hardware

Hardware

MacBook Pro 14″, M3 Pro

Daily driver

Main machine for terminal work, IaC, and dashboards. 36GB RAM keeps multiple SSH sessions, a browser full of console tabs, and Docker comfortable at once.

Runs Terraform plans and a local k8s cluster without breaking a sweat.

Dual 27″ monitors

Desk

One screen for terminal and editor, the other for CloudWatch dashboards, Grafana, and whatever's on fire that day.

YubiKey 5C

Security

Hardware key for AWS root account and IAM Identity Center MFA — no phone-based 2FA on anything that touches production.

Keychron mechanical keyboard

Input

Low-profile board, brown switches. Survives long stretches of typing out Terraform and YAML without complaint.

Logitech MX Master 3

Input

Pairs over Bluetooth across the laptop and a spare desktop used for load-testing and long-running jobs.

~/dev

Editor & terminal

VS Code

Editor

Primary editor for Terraform, YAML, and Python — HashiCorp Terraform, AWS Toolkit, and Remote-SSH extensions do most of the heavy lifting.

iTerm2

Terminal

Split panes for tailing logs in one pane while running commands in another — profiles set up per AWS account to avoid mixing prod and staging.

zsh + Powerlevel10k

Shell

Prompt shows git branch, k8s context, and active AWS profile at a glance — the fastest way to avoid running the wrong command in the wrong account.

tmux

Multiplexer

Keeps sessions alive over SSH to bastion hosts and jump boxes, so a dropped VPN connection doesn't kill a long-running command.

AWS CLI v2

CLI

Day-to-day resource inspection and scripting — paired with aws-vault so credentials never sit unencrypted on disk.

kubectl + k9s

Kubernetes

k9s for a fast terminal UI over clusters; raw kubectl when a script needs exact, scriptable output.

~/cloud

Cloud & AWS

AWS (multi-account)

Platform

Primary cloud — EC2, S3, VPC, RDS, and IAM day to day, spread across separate dev, staging, and prod accounts under an AWS Organization.

IAM Identity Center

Access

Single sign-on across every AWS account — no long-lived IAM user keys floating around.

Amazon EKS

Orchestration

Managed Kubernetes for containerized workloads — node groups sized and scaled through Terraform, not clicked together in the console.

AWS Systems Manager

Ops

Session Manager instead of SSH keys for shell access to instances, plus Patch Manager for keeping fleets up to date.

Amazon CloudWatch

Observability

Logs, metrics, and alarms baked into every stack before it ships — the first stop when something's slow or noisy.

~/iac

Infrastructure as Code & config

Terraform

IaC

Primary tool for provisioning across every environment — modules per service, remote state in S3 with a DynamoDB lock table.

AWS CloudFormation

IaC

Used where a stack needs tight native AWS support — CDK-generated templates included.

Ansible

Config mgmt

Configures EC2 fleets after Terraform provisions them — package installs, users, and application config.

Packer

Image builds

Bakes golden AMIs with the base config and monitoring agents already installed, so instances boot ready to work.

~/cicd

CI/CD & containers

GitHub Actions

CI/CD

Pipelines for infra changes — plan on pull request, apply on merge, with required approvals before anything touches prod.

Docker

Containers

Local builds and testing before images ship to a registry and out to EKS.

Jenkins

CI/CD

Still runs a handful of legacy pipelines that predate the move to GitHub Actions — on the list to migrate.

Helm

Kubernetes

Templates manifests for EKS deployments so environment differences live in values files, not copy-pasted YAML.

~/monitoring

Monitoring & collaboration

Grafana + Prometheus

Observability

Dashboards for cluster and service-level metrics, scraped straight out of EKS workloads.

Datadog

Monitoring

Unified metrics, logs, and traces across accounts — the first place to check when an alert fires at 2am.

PagerDuty

On-call

Incident alerting and escalation, wired into CloudWatch and Datadog alarms.

Jira

Tasks

Sprint tracking and change tickets — every infra change traces back to a ticket.

Confluence + Slack

Docs & comms

Runbooks and architecture docs live in Confluence; day-to-day coordination and alert noise live in Slack.

This list changes more often than it should — check back for updates, or ask on the contact page if something here has been swapped out.
