Uses

The hardware, terminal setup, and day-to-day tools behind the writing, code, and recordings on this site — the boring stack that keeps everything running.

jordan@studio:~/uses$ cat README.md
hardware editor & terminal browser writing & design podcast & recording productivity
~/hardware

Hardware

MacBook Pro 14″, M3 Pro

Daily driver

Main machine for writing, development, and editing. 36GB RAM keeps it comfortable running a browser, editor, and audio tools at once.

Battery life survives a full day of travel between talks.

27″ 4K monitor

Desk

External display for the home setup, docked next to the laptop for a two-screen writing and coding layout.

Keychron mechanical keyboard

Input

Low-profile mechanical board, brown switches. Swapped in after one too many long writing sessions on a laptop keyboard.

Logitech MX Master

Input

Ergonomic mouse that also drives the second machine over Bluetooth when switching between desk and stage setups.

iPhone + tripod rig

Mobile

Used for quick voice memos, on-stage backup recordings, and behind-the-scenes clips for the blog.

~/dev

Editor & terminal

VS Code

Editor

Primary editor, kept lean — a handful of extensions, a synced settings file, and a theme that matches this site's dark palette.

Warp

Terminal

Terminal emulator with command blocks and AI-assisted lookups for the commands I always forget the flags to.

zsh + oh-my-zsh

Shell

Shell of choice, with a minimal prompt theme showing just the path and git branch — no clutter.

tmux

Multiplexer

Keeps long-running sessions alive over SSH for anything running on a remote box or a Raspberry Pi.

GitHub + GitHub CLI

Version control

All projects live here; the CLI handles PRs and issues without leaving the terminal.

~/browser

Browser

Arc

Daily

Spaces keep work, writing research, and personal tabs cleanly separated without twelve windows open at once.

1Password

Extension

Password manager and the thing that makes two-factor logins on stage demos actually bearable.

uBlock Origin

Extension

Non-negotiable. Faster pages, fewer trackers.

~/writing

Writing & design

iA Writer

Drafting

Where blog posts and notes start life — plain markdown, no formatting distractions until the piece is ready.

Figma

Design

Slide decks for talks and quick mockups for this site's layout before they become actual HTML and CSS.

Grammarly

Editing

A second pass on longer posts, mostly for catching the typos a tired brain skips over.

~/podcast

Podcast & recording

Shure SM7B

Microphone

Broadcast-standard dynamic mic for the podcast and any recorded talks — flat, warm, forgiving of a noisy room.

Focusrite Scarlett 2i2

Interface

Clean preamp and the bridge between the mic and the laptop for every episode.

Riverside.fm

Recording

Records local high-quality tracks for remote guest interviews, so a shaky connection never ruins the audio.

Descript

Editing

Edits episodes by editing the transcript — cuts filler words and dead air in a fraction of the usual time.

~/workflow

Productivity

Notion

Notes

Talk outlines, blog drafts, and a running backlog of ideas that may or may not ever become posts.

Linear

Tasks

Tracks anything with more than one step — site changes, talk prep, podcast production.

Raycast

Launcher

Replaces Spotlight for app switching, clipboard history, and a handful of small custom scripts.

This list changes more often than it should — check back for updates, or ask on the contact page if something here has been swapped out.
